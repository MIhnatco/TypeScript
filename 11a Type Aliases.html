Type Aliases

Type aliases in TypeScript allow us to create custom names for types, making our code more readable and expressive. In other words, type alias is just providing another name for something that already exists.

Syntax:

                type <AliasName> = <Type>;

Type aliases are created using the type keyword, followed by the alias name and the type it represents.

Example:

            
                type PersonName = string;

                const myName: PersonName = "Jayson";
                const alsoMyName: string = "Jayson";

When you put your mouse over 'myName', TypeScript says it's a string because we told it it's a string. Because 'PersonName' is purely for us as developers, so we can document our code. And even if we put our mouse over 'PersonName', it telss us that the type is a string. (It's strictly for developer documentation to create code that is more self-documenting).

Example:

In real code, you have a lot of lines of code...and you don't need to see the function's code to understand it. SO the first function says..it will 'print' something, 'thing' is definitely going to be anything. Who knows what that is? And at least we know it is a string...So we can call the function, but it might not do what we think it does. 

        
    //bad 
    function print(thing: string){
        console.lot(`Name is ${thing}`)
    }

    print("a house")  //Name is a house.

Now if we rewrite the function using better naming and our type alias, this is going to become a lot more clear. 

        
            //good 
    function printName(name: PersonName){
        console.log(`Name is ${name}`)
    }

Now if we collapse this function here, we can see we have a function named 'printName' and we must supply a 'name' and that name has to be a 'PersonName'. And if we don't know what a "PersonName" is , then we can just mouse over that and we can see .. it's a string. And we know for sure.. it needs to be a person name.
So we use a type alias instead of a 'string'. A 'string' maybe the name or could be a name of an address. It could be a name of a business. It could be a name of a fruit.

Creating Aliases for Union Types | - or

Syntax:

        
          type <AliasName> = <Type1> | <Type2>;

Type aliases can be used to create custom names for union types, which represent a value that can be one of several types. 

      
        Example: type Pet = "dog" | "cat" | "bird";

        // Using the | we are saying our parameter is a string or number:
        function printStatusCode(code: string | number) {
          console.log(`My status code is ${code}.`)
        }

Creating Aliases for Intersection Types

Syntax:

        
          type <AliasName> = <Type1> & <Type2>;

Type aliases can also be used to create custom names for intersection types, which represent a value that must satisfy multiple types. 

    
      Example: type Employee = { name: string } & { id: number };
