Why TypeScript?

Why should we learn TypeScript when we already have JavaScript?

To gain a thorough appreciation of the advantages TypeScript offers, let's begin by examining JavaScript. You don't need to possess any prior knowledge of JavaScript to follow along.
Beneath, we encounter a function named 'popup' that serves to display a pop-up dialog to the user. This 'popup' affords customization of its message and provides various 'options'.
All seems well at first glance; however, we encounter a stumbling block in the very first line of code. 

    
function popup(msg, options){

}
  • The question that arises is: What exactly is 'msg'? Presumably, it contains text data, as that would be the most logical deduction.
  • But what about 'options'? What options are at our disposal, and what if we don't set any 'options'? And what if we happen to define an excessive number of 'options'? These are all questions left unanswered. In essence, we possess scant information regarding this function beyond its name, 'popup,' and the presence of 'msg' and 'options' as parameters.

Now, here we have the full code for the 'popup' function. We will focus only at these four lines on the right.

These four lines set up the default 'options'. If we try to use this function without any 'options', then we'll get these defaults. This is useful information because now we know the available options.

We know there is a 'kind' options and we know that the default is 'info'. This is probably what kind of pop-up we can display since we can make information. Maybe we can make air pop-up or warning pop-ups. We don't know because there isn't any documentation for this example function. So we would need to read the code further to figure out if there are other kinds of pop-ups and how to use them.

We also have this 'timeout' options, which likely means how long the message is displayed on screens. The number is 5000, which seems like a lot. But are those in minutes, second, miliseconds? They're probably to be in milliseconds becaue 5000 seconds is over an hour.

This is an imaginary scenario so we don't really need to dig further into the code. It's safe to assume that the 'timeout' is in milliseconds. If you wanted, this is fine as a consumer of the function. We'll actually be able to use this code and probably get an information.

But what if we were the person who actually had to write and maintain this function. We could set our defaults here in the beginning and then we write the functionality . This makes sense because once we have the option set, we can make the 'popup'. Remember, how we weren't exactly sure what kinds of popups were available and we didn't know if 5000 was in minutes or milliseconds. Well, there needs to be code to check that. And that's what these lines mark with red doing.

So we have our five lines, that's at the defaults which is fine, because we want to be able to use this function with defaluts. And then we have 23 lines to check everything else. This is called 'boilerplate' code. It helps us get for we need to go, but doesn't actually do the thing we're trying to do which is to just make pop-up... To make pop-up, the bottom is where our code is actually suppsed to go. There is nothing yet.

We should also write some tests to make sure that our function properly rejects broken options and invalid messages.

For example, the test will check to make sure that the function works as expected if we give it a negative timeout, duration, or if we don't give a message or if we try to make it unsupported kind of pop-up. The function should produce an error in all these situations.
If we add up the code, then we get 59 lines of code. Remember, we haven't written any functionality yet to actually make a pop-up, and we already have 23 lines of boilerplate code and six test cases.

There is no functional code here and there are zero tests to make sure that the pop-up actually works.

The test cases are there just to make sure we can actually call this function properly.
This code is alsy only useful for this one function. So none of these 59 lines can tribute to any other part of the code base.
