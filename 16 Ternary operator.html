Ternary operator

The ternary operator is a shorthand way to express conditional logic. It allows you to make decisions and choose between two values or expressions based on a condition. The ternary operator is useful for concise conditional assignments.

Syntax and Usage of the Ternary Operator:

The syntax of the ternary operator is as follows: 

    
        condition ? expressionIfTrue : expressionIfFalse
  • condition: A condition that evaluates to either true or false.
  • expressionIfTrue: The value or expression to return if the condition is true.
  • expressionIfFalse: The value or expression to return if the condition is false.

Here's an example: 

    
        let isRaining = true;
        let weatherMessage = isRaining ? "Don't forget your umbrella!" : "Enjoy the sunshine!";

In this example, isRaining is true, so the ternary operator returns "Don't forget your umbrella!"

Ternary Operator vs. if Statements:

The ternary operator is a more concise way to express simple conditional logic compared to if statements. However, it's not suitable for complex conditions with multiple branches. if statements are still more flexible in such scenarios.

Exercise: Temperature Message

Pseudo-Code: You are building a temperature message program. Given a numeric temperature in degrees Celsius, you should determine the corresponding message based on the following criteria:

  • If the temperature is below 0°C, display "It's freezing!"
  • If the temperature is between 0°C (inclusive) and 15°C (exclusive), display "It's cold."
  • If the temperature is between 15°C (inclusive) and 25°C (exclusive), display "It's pleasant."
  • If the temperature is 25°C or higher, display "It's warm."

Use the ternary operator to implement this. You should assign the temperature message to a variable based on the temperature value.

Mnemonic Devices:

  • Think of the ternary operator as a mini decision maker: it quickly picks between two options based on a condition.
  • The ? symbol resembles a question mark, representing the condition you're asking.

It's particularly useful for simple one-liner conditions.
