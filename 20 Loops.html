Loops

Loops are a fundamental concept in programming, and they serve several crucial purposes:

  • Repetition: Loops allow you to execute a block of code multiple times, often with slight variations. This is essential for automating repetitive tasks, which can save time and reduce the risk of errors in manual repetition.
  • Iterating over data:Loops are commonly used to iterate over collections of data, such as arrays, lists, or databases. They enable you to process each element of the data one at a time.
  • Dynamic control flow:Loops provide dynamic control flow by allowing your program to adapt to changing conditions. For example, you can use loops to respond to user input, process data until certain conditions are met, or perform tasks based on real-time events.
  • Efficiency: Loops are more efficient than writing the same code multiple times. They enable you to reuse a block of code and change only the parts that need variation, leading to more concise and maintainable code.
  • Generating sequences:Loops can be used to generate sequences of numbers or other data, making them invaluable for tasks like counting, creating patterns, or generating random values.
  • Data manipulation:oops are essential for data manipulation, including sorting, filtering, and transforming data. They provide a way to apply operations consistently to multiple data items.
  • Algorithm implementation: Many algorithms, from basic searching and sorting to complex machine learning algorithms, rely on loops to iterate through data and perform calculations.
  • Creating dynamic interfaces:Loops are used to generate dynamic content in user interfaces, such as populating dropdown menus, creating tables, or displaying lists of items.
  • Handling events: Loops can be used to manage event handling in applications. For example, you can continuously check for user input or respond to real-time events like mouse movements.

for loop

A 'for' loop is a control structure that allows you to execute a block of code repeatedly based on a specified condition. It's commonly used when you know the number of iterations you want to perform or when you need to iterate over a collection of values, like an array.

Syntax of a 'for' loop.

The basic syntax of a 'for' loop in TypeScript is as follows: 

    
        for (initialization; condition; increment){
            //code to be executed in each iteration
        }
  • Initialization: this is where you initialize a loop control variable.
  • Condition: the loop continues as long as this condition is true.
  • Increment: the loop control variable is modified in this part of the loop. The loop control variable is often used to track the current iteration or index. You can increment or decrement it in each iteration to control the loop's behavior.

Simple example: The code is a "for" loop in TypeScript that iterates through a range of numbers from 0 to 5 (inclusive) and logs each number to the console. 

    
        for (let i = 0; i <= 5; i++){
            console.log(`${i}`)
        }
  • 'let i = 0': Initialization, where 'i' s declared and set to 0
  • 'i <= 5': Condition, the loop will continue as long as 'i' is less than or equal to 5.
  • 'i++' Increment, 'i' is incremented by 1 after each iteration.
  • 'console.log('${i}'): Inside the loop, this line logs the current value of 'i' to the console.

This loop demonstrates how you can use a "for" loop to iterate over a range of numbers and perform actions for each value in the range. In this case, it's logging the numbers from 0 to 5.

Let's consider a real-life example, where you might use a 'for' loop: calculating the total cost of items in a shopping cart. In this scenario, you want to iterate through each item in the cart and sum their prices to get the total cost. 

    
        //Step 1: Create a shopping cart with items
            const shoppingCart: {name: string; price: number}[] = [
                { name: "Product A", price: 10.99},
                { name: "Product B", price: 5.49},
                { name: "Product C", price: 7.99}
            ]
        //Step 2: Initialize the total cost
            let totalCost: number = 0;

        //Step 3 and 4: Use the 'for' loop to iterate through the shopping cart and calculate the total cost
            for (let i = 0; i < shoppingCart.length; i++){
                totalCost += shoppingCart[i].price
            }

        //Step 5: The totalCost variable now holds the total cost of items in the cart 
            console.log("Total Cost: $" + totalCost.toFixed(2))

In this code, the "for" loop iterates through each item in the shopping cart, and the prices of the items are added to the totalCost variable. After the loop, the total cost is calculated and displayed. This is a practical use of a "for" loop for calculating totals in a real-life scenario.

Mnemonic Devices:

  • Think of a "for" loop as a repetitive task performed "for" a specific number of times.
  • Use initialization, condition, and increment to remember the structure of a "for" loop.

'break' statement

The break statement is used within a loop, including a "for" loop, to prematurely exit the loop before it reaches its normal termination condition. When break is encountered, it immediately terminates the loop and program control is transferred to the first statement after the loop.

In the context of 'for' loop, here's how 'break' is typically used: 

    
        for (let i = 0; i < 10; i++){
            if(i === 5){
                break; //exit the loop when i equals 5
            }

            console.log(i)
        }

In this example, the loop iterates through values of 'i' from 0 to 9. However, when 'i' becomes equal to 5, the 'break' statement is encountered, causing the loop to texit immediately. As a result, only values from 0 to 4 are printed to the console.

Key points about using break in a "for" loop:

  • Premature Termination: break provides a way to prematurely exit the loop, even if the loop's termination condition hasn't been met. It's often used to respond to specific conditions within the loop.
  • Use Cases:Common use cases for break include early exit based on certain conditions, implementing search or matching algorithms, and stopping the loop in response to user input.
  • Control flow:Once the break statement is executed, the loop is terminated, and control is passed to the first statement after the loop, skipping any remaining iterations.
  • Nested Loops:When dealing with nested loops (loops within loops), break will exit the innermost loop it is placed in. To exit an outer loop, you may need to use additional logic or labels.

'continue' statement

The continue statement is used within a loop, including a "for" loop, to skip the current iteration and move to the next one. Unlike the break statement, which exits the loop entirely, continue allows you to bypass the current iteration and proceed to the next iteration.

In the context of a 'for' loop, here's how 'contiune' is typically used: 

    
        for (let i = 0; i < 10; i++){
            if (i % 2 === 0){
                continue;  //skip even numbers
            }

            console.log(i)
        }

In this example, the loop iterates through values of 'i' from 0 to 9. When 'i' is an even number, the 'continue' statement is encounterd, and that iteration of the loop is skipped. The loop then proceeds to the next value of 'i'.

Key points about using continue in a "for" loop:

  • Skipping Iterations: continue skips the current iteration and proceeds to the next iteration. It's often used to bypass certain iterations based on specific conditions within the loop.
  • Use Cases: Common use cases for continue include filtering or processing specific elements in an array or sequence, and executing different code for different iterations.
  • Control Flow: When continue is executed, the loop jumps back to the loop's increment statement (e.g., i++), and the next iteration begins. Any code after the continue statement within the current iteration is skipped.

Using continue allows you to control the flow of your "for" loop, skipping specific iterations as needed, while continuing to iterate through the rest of the elements.

while loop

A "while" loop is a control structure that repeatedly executes a block of code while a specified condition is true. It's particularly useful when you want to keep executing a block of code until a certain condition is met.

Syntax of a 'while' loop

The basic syntax of a 'while' loop in TypeScript is as follows: 

    
        while (condition){
            //code to be executed as long as the condition is true
        }

The 'condition' is a Boolean expression that determines whether the loop should continue or terminate. If the condition is false when the loop is first encounterd, the loop's block of code is never executed.

To use 'while' loop effectively, you often need to initialize and update variables within the loop block to control the loop's behavior.

Use cases for 'while' loops

'while' loops are commonly used in scenarios where the number of iterations is not known in advance but is determined by some condition. Use cases include input validation, interactive applications, and event handling.

Here's a simple example of a 'while' loop that counts from 1 to 5: 

    
        let count = 1;
        while (count <= 5){
            console.log(count);
            count++;
        }

This loop continues executing as long as the condition 'count <= 5' is true. The variable 'count' is updated in each iteration to control the loop.

Mnemonic Devices:

  • Think of a "while" loop as a continuous process that repeats while a condition holds true.
  • Use the while keyword to remember the loop's structure.
