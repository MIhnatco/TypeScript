Optional Fields

In TypeScript, you can declare fields in interfaces or object types as optional by adding a '?' symbol after the field name. This indicates that the field may or may not be present when creating objects of that type. Optional fields are useful when you want to allow flexibility in the shape of objects without requiring every field to be provided.

Here's a basic example of optional fields in TypeScript:

    
        interface Person {
            name: string;
            age?: number; //age is an optional field
        }

        const person1: Person = {
            name: "Alice"
        }

        const person2: Person = {
            name: 'Bob', 
            age: 25;
        }

In this example, the Person interface defines two fields, name and age. However, age is marked as optional with ?. As a result:

  • You can create an object with only the name field (as shown in person1).
  • You can also create an object with both name and age fields (as shown in person2).

Assignment: User profile

Imagine you're developing a user profile system for a website. Users can create profiles with various pieces of information, some of which are optional. Your task is to create a TypeScript interface for a user profile that captures both required and optional fields.

  1. Define an interface UserProfile that includes the following fields:
    • username (a required string)
    • email (a required string)
    • age (an optional number)
    • location (an optional string)
    • bio (an optional string)
  2. Create a function printUserProfile that takes an argument of type UserProfile and prints the user's profile information.
  3. Create two user profiles, one with all required fields and another with some optional fields, and use the printUserProfile function to display their profiles.
  4. Handle the optional fields gracefully in the function so that it doesn't display anything for those fields if they are not provided.

Your solution should demonstrate the usage of optional fields in the UserProfile interface.

Optional fields provide flexibility when dealing with objects that may not have all properties defined, which is common in real-world scenarios. They help you express this flexibility in the type system while still enforcing type safety where needed.
