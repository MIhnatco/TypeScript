Variable and Basic types

A variable is a named memory location that can hold a value.
Variables can be used to store a wide range of data types, such as numbers, strings, and arrays.
A variable is declared by specifying its name, data type, and optionally an initial value. Once a variable is declared, it can be read potentially updated in other parts of the program.

Variable Declarations in TypeScript:

In TypeScript, variables are declared using the let or const keyword. As discussed earlier, const is used for constants that cannot be reassigned, while let is used for variables that can change their values.
Here's an example: 

        
            let age: number = 30; // Declaring a variable 'age' of type 'number'
            const name: string = "Alice"; // Declaring a constant 'name' of type 'string'

Data Types in TypeScript:

TypeScript, as a statically typed language, allows you to specify the data type of a variable. Some common data types in TypeScript include:

  • number for numeric values
    • const amount = 10
    • const fraction = 10.2
    • const oneThousand = 10e3
  • string for text values
    • const courseName = 'TypeScript'
    • const courseName = "TypeScript"
  • boolean for true/false values
    • const yes = true;
    • const no = false;
  • Completely missing:
    • const missing = undefined;
  • We know it's missing:
    • const empty = null;

For example: 


    let quantity: number = 5;
    let productName: string = "Laptop";
    let isAvailable: boolean = true;
    let scores: number[] = [90, 85, 78];
    let person: object = { name: "Bob", age: 25 };
    let dynamicValue: any = "This can be anything";

Type Inference

TypeScript also provides type inference, which means it can often determine the data type of a variable without explicit annotation.
For instance: 

    
        let temperature = 25; //TypeScript infers the type 'number'
        let greeting = "Hello";  //TypeScript infers the type 'string'

Type inference reduces the need for explicit type annotations, making your code more concise.

Type Annotations

While TypeScript can infer types, you can also provide explicit type annotations to make your code more explicit and self-documenting. Type annotations are especially useful for function parameters and return types. These annotations help the TypeScript compiler catch type-related errors during development. 

        function add(a: number, b:number): number {
            return a + b
        }

Mnemonic Devices:

Think of variables as containers that hold different types of data. You label each container with a data type to specify what kind of data it can hold.

Type inference is like magic - TypeScript often guesses the variable's type for you, but you can always be explicit with type annotations to remove any uncertainty.
