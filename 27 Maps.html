Data Structure: Map

  • Maps are a data structure that stores key-value paris, providing efficinet data retrieval.
  • Each 'key' in a Map is unique, and it maps to a specifi value.
  • They differ from regular objects by allowing any data type as keys and maintaining key order.
  • Unlike regural objects, Maps can use a wide range of data types as keys, including objects, functions, and primitive values like strings and numbers.

How Maps differ from Objects:

While regular JavaScript objects can also store key-value pairs, Maps have several advantages:

  • Keys can be of various data types, making Maps more flexible.
  • Map keys maintain their insertion order, which means the order in which you add key-value pairs is preserved.
  • Maps are more suitable for scenarios where you need to associate data and perform key-based operations efficiently.

Key Characteristics of Maps:

  • Keys in a Map are unique. If you add the same key multiple times, it will only store the latest associated value.
  • Maps provide methods to add, access, and manipulate key-value pairs.
  • Maps maintain the insertion order of keys, which is beneficial for certain operations.

Adding Key-Value Pairs:

You can add 'key-value' pairs to a Map using 'set(key,value)' method.
Here's how it works: 

    
        type Name = string;
        type Score = number;

        const testScores: Map<Name, Score> = new Map()

        testScores.set("Alice", 96);
        testScores.set("Bob", 88);
        testScores.set("Carol", 92);


//To find out the size of the Map 
console.log(testScores.size)  //Output: 3

Accessing Values by Keys:

To retrieve the value associated with a specific key, you can use the 'get(key)' method: 

    
const Alice's score = testScores.get('Alice')
const Bob's score = testScores.get('Bob')
const Carol's score = testScores.get('Carol')

Updating Key-Value Pairs:

If you want to update the value associated with an existing key, you can use the 'set' method again with the same key. This will replace the old value with the new one: 

    
testScores.set("Alice", 99);  //Updates the 'Alice' value to 99

The key remains the same, but the value is updated.

Adding multiple key-value pairs at once:

You can initialize a Map with multiple key-value pairs by passing an iterable, such as an array of arrays containing key-value pairs: 

    
        const myMap = new Map([
            ['name', "Alice"], 
            ['age', 25], 
            ['city', "New York"]
        ])

This creates a Map with three key-value pairs.

Handling Non-existent keys:

When attempting to access a key that does not exist in the Map, the 'get' method returns 'undefined'. It's a good practice to check if a key exists in the Map before attempting to access it.

To check if the 'key' exists in a Map() we can use 'has' method: 

    
        myMap.has('age')

Deleting key-value pairs

To remove a key-value pair from a Map, you can use the 'delete(key)' method:

    
        myMap.delete("age");   //Deletes the 'age' key-value pair

Reset the Map

The 'Map.clear()' method is used to remove all key-value pairs from a Map, effectively emptying it. This method does not return any value, and it operates directly on the Map, modifying it in place. 

    
        myMap.clear()

Iterating over Maps

Using 'for...of' loop:

TypeScript provides a convenient way to iterate over the key-value pairs in a Map using the for...of loop.
Here's how it works:

    
        const myMap = new Map([
    ["name", "Alice"],
    ["age", 25],
    ["city", "New York"],
]);

for (const [key, value] of myMap) {
    console.log(`Key: ${key}, Value: ${value}`);
}

Using the forEach Method:

Maps have a forEach method that can be used to iterate over the key-value pairs. It takes a callback function as an argument: 

    
        myMap.forEach((value, key) => {
            console.log(`Key: ${key}, Value: ${value}`);
        });

The forEach method iterates through the Map and invokes the provided callback function for each key-value pair.

Iterating Over Keys and Values Separately:

If you need to iterate over keys or values alone, you can do so using the keys() and values() methods of the Map. For example: 

    
        for (const key of myMap.keys()) {
            console.log(`Key: ${key}`);
        }
        for (const value of myMap.values()) {
            console.log(`Value: ${value}`);
        }

Iterating Over Entries:

The entries() method returns an iterable of key-value pairs, making it similar to the default behavior of the for...of loop. For example: 

    
        for (const [key, value] of myMap.entries()) {
            console.log(`Key: ${key}, Value: ${value}`);
        }

You can use Map iteration to perform various tasks, such as filtering data, searching for specific values, or transforming the data as needed.

Requirements:

  1. Create an interface named Student with the following properties:

    • name (string): The name of the student.
    • grades (Map): A Map that stores subject names (string) as keys and the corresponding grades (number) as values. Each student can have multiple subjects and grades.
  2. Implement a class called StudentRecord that implements the Student interface. This class should have the following methods:
    • constructor(id: number, name: string): Initializes a student's ID and name. Initialize an empty Map for grades.
    • addGrade(subject: string, grade: number): Adds a subject and its corresponding grade to the student's record. If the subject already exists, update the grade.
    • getAverageGrade(): Calculates and returns the average grade for the student. Use the grades stored in the Map.
    • printStudentInfo(): Displays the student's ID, name, and their grade information (subject and grade).

Create at least two instances of the StudentRecord class, add grades for different subjects to each student, and calculate and display their average grades and information.

Ensure that you handle student data efficiently using the provided class and interface. Make sure the code is well-organized and adheres to good TypeScript practices.

Requirements:

  1. Create an interface named "Book" with the following properties:
    • 'title'(string): The title of the book.

    • 'ISBN'(string): The Internation Standard Book Number.
    • 'isAvailable'(boolean): Indicates if the book is available for borrowing.
  2. Implement a class named "Library" that manages the library catalog. This class should have the following methods:

    • 'listCatalog()'': Displays the current catalog of books, including their title, author, ISBN, and availability status.
  3. Implement a class named User that represents library users. Each user should have a name and a borrowedBooks Map that associates borrowed books (by title) with the due date. A user can borrow multiple books.
  4. Create at least three instances of the Book class and add them to the library catalog. Simulate user interactions where users borrow and return books.
  5. Ensure that the program handles unique ISBNs, keeps track of book availability, and provides clear information about the catalog and users' borrowed books.
