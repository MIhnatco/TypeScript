Modules

Modules allow you to organize and encapsulate your code into reusable and maintainable units. They are essential for structuring large-scale applications and facilitating code separation and reusability.

Modules in TypeScript are a way to structure and organize code. They allow you to group related functions, classes and variables into a single unit. Modules help in avoiding naming conflicts, enhancing code maintainability, and making your code more organized and reusable.

Exporting and Importing Modules

  • Exporting: In TypeScript, you can mark specific elements within a module as exportable using the 'export' keyword. These elements become accessible to other modules.
  • Importing: You can use the 'import' statement to bring exported elements from one module into another module, allowing you to use those elements in your code.

Modules can be organized in files. Each TypeScript file can be a module, and the structure of your code should reflect the organization of your module. This facilitates code navigation and organization.

Examples:

  • Creating and exporting a module: 
        
        //mymodule.ts 

        export function greet(name: string): string {
            return `Hello, ${name}!`;
        }
  • Importing and using a module: 
        
        //app.ts 
    import {greet} from './mymodule';

    const message = greet("Alice");

    console.log(message);  //Output: Hello, Alice!

Understanding modules is crucial for building organized, maintainable, and scalable TypeScript applications.
