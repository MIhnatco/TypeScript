enum type

In TypeScript, an 'enum' (short for 'enumeration') is a data type used to define a set of named constants, which represents a set ofo related values. Enums make your code more readable, and self-documenting by providing meaningful names for values, rather than using raw number or strings. They are particularly useful when you have a fixed and well-defined set of options or values.

Definig an Enum:

To define an 'enum', use the 'enum' keyword followed by a name for the enum and a set of named constants enclosed in curly braces. Each constant should have a name and an optional associated value. The associated value can be of any type (number or string). 

    
        enum Days {
            Sunday,   //0
            Monday,   //1
            Tuesday,  //2
            Wednesday,  //3
            Thursday,   //4
            Friday,     //5
            Saturday    //6
        }

In the example above, we've defined an enum named "Days" with seven named constants. By default, each constant is assigned a numeric value starting from 0, and subsequent values are incremented by 1.

Using an Enum:

You can use the enum in your code to represent values and provide meaningful names. For example: 

    
        let today: Days = Days.Wednesday;
        console.log("Today is " + today);   //Output: Today is 3

Reverse Mapping:

Enums in TypeScript support reverse mapping, which means you can convert from a numeric value or string to the corresponding enum member name. For example: 

    
        let dayName: string = Days[3]; //This returns "Wednesday"

This is particularly useful when you need to display or work with human-readable names based on enum values.

Custom Values for enum members:

You can also assign custom values to enum members:

    
        enum Color {
            Red = "#f00",
            Green = "#0f0", 
            Blue = "#00f"
        }

Here, we've defined an enum "Color" with custom string values representing color codes.

Reverse mapping is primarily designed for use with numeric enum values and may not work as expected with custom values for enum members. When you have custom values for enum members, TypeScript doesn't generate a reverse mapping from values to member names automatically. Instead, reverse mapping is primarily applicable when using the default incremental numeric values.

Use cases for Enums:

Enums are commonly used whne you have a set of options or categories, that need to be represented in your code, such as days of the week, colors, status codes, or error types. They provide a structured and type-safe way to work with these values.

Enums help prevent accidental misuse of values and improve code quality by making it clear what each value represents. They also enhance code maintainability and readability.
