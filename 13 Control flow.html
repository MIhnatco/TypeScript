Control Flow

Conditional statements in TypeScript allow you to execute different blocks of code based on whether a condition is true or false. These statements help your program make decisions.

The 'if' statement

The 'if' statement is one of the most fundamental conditional statements in TypeScript. It allows you to execute a block of code if a specified condition is 'true'. 

    
        let age = 18;
        if (age >= 18){
            console.log("You are an adult.")
        }

In this example, the code inside the 'if' block will only run if the condition 'age >= 18' is true.

The 'else' statement

You can use the 'else' statement to provide an alternative code block to run if the 'if' condition is 'false'. 

    
        let temperature = 25;
        if (temperature > 30){
            console.log("It's hot outside.")
        } else {
            console.log("It's not too ht outside.")
        }

In this example, if the temperature is greater than 30, the first message is displayed. Otherwise, the message in the 'else' block is shown.

The 'else if' statement

The 'else if' statement is used when you have multiple conditions to check in a controlled and organized manner. It comes after the initial 'if' statement and before the optional 'else' statement. 

    
        let score = 75

        if (score >= 90){
            console.log("You got an A.")
        } else if (score >= 80){
            console.log("You got a B.")
        } else if (score >= 70){
            console.log("You got a C.")
        } else {
            console.log("You need to improve your score.")
        }

In this example, the code will check the score against multiple conditions. If the score is greater than or equal to 90, it prints "You got an A." If not, it moves to the next else if block and checks if the score is greater than or equal to 80, and so on.

The else if statement allows you to create a chain of conditions that are evaluated in order. As soon as one condition is met, the corresponding block of code is executed, and the remaining conditions are skipped.

Comparison operators

Comparison operators such as ==, !=, <, >, <=, and >= are used to compare values in conditions.

Equality Operators == & ===

There are two operators for checking equality in Typescript. One is (==) known as an equality operator or loose equality operator. The other one is (===) strict Equality operator.

Not Equal Operators != & !==

!= operator checks the un equality of two operands. It is similar to == except it returns true if operands are not equal. !== operator is similar to === except it returns true if operands are not equal.

Less than (<)

The Less than (<) comparison operator checks if the left operand is less than its right operand. If yes then it returns true otherwise returns false

Greater than (>)

The greater than > operator checks if the value of the left operand is greater than the right operator. If yes then it returns true else false.

Less than or equal (<=)

The less than or equal (<=) operator returns true if the left operand is less than or equal to its right operand. else it evaluates to false.

Greater than or equal (>=)

The Greater than or equal (>=) relational operator returns true if its left operand is greater than or equal to its right operand else it returns false. 

    
let x = 10;
if (x > 5) {
  // Code to execute if x is greater than 5.
}

console.log(x <= 20)    //true
console.log(x <= 15)    //true
console.log(x <= 10)    //true
console.log(x <= 5)     //false

console.log(x >= 20)    //false
console.log(x >= 15)    //false
console.log(x >= 10)    //true
console.log(x >= 5)     //true

Logical operators

Logical operators are used to combine two or more conditions. Logical operators retun a Boolean value.
Assume the value of variable A is 10 and B is 20.

Operator Description Example
&& (And) The operator returns true only if all the expressins specified return true (A > 10 && B > 10) is False
|| (Or) The operator returns true if at least one of the expressions specified return true (A > 10 || B > 10) is True
! (NOT) The operator returns the inverse of the expression's result. !(A > 10) is True

Short-circuit Operators (&& and ||)

The && and || operators are used to combine expressions. The && operator returns true only when both the conditions return true. Let us consider an expression âˆ’ 

    
        var a = 10 
        var result = ( a<10 && a>5)

In the above example, a < 10 and a > 5 are two expressions combined by an && operator. Here, the first expression returns false. However, the && operator requires both the expressions return true. So, the operator skips the second expression.

The || operator returns true, if one of the expressions returns true. For example:

    
        var a = 10 
        var result = ( a < 5 || a >10)

In the above snippet, two expressions a > 5 and a < 10 are combined by a || operator. Here, the first expression returns true. Since, the first expression returns true, the || operator skips the subsequent expression and returns true.

Due to this behavior of the && and || operator, they are called as short-circuit operators.

Mnemonic Devices:

  • Think of the if statement as a gatekeeper: it lets code through only if a certain condition is met.
  • The else statement is the backup plan for when the if condition is not satisfied.

Here are a few key points to keep in mind when using else if:

  • Conditions are evaluated from top to bottom. Once a condition is true, the associated block of code is executed, and the rest of the else if and else blocks are skipped.
  • You can have multiple 'else if' blocks, creating a hierarchy of conditions.
  • You can have an else block at the end to provide a default action in case none of the conditions are met.
  • The else if statement is an effective way to handle complex decision-making scenarios and offers a structured approach to handling various cases in your code.

These statements are essential for building programs that can make decisions and respond to different situations.
