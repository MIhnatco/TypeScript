Type Annotations

Type annotations in TypeScript are used to explicitly specify types for variables, functions, objects, and more. They are not mandatory in TypeScript, but they help the compiler in checking types and avoiding errors when dealing with data types.

Type annotations are declared using a colon (:) followed by the name of the type, and they come after the name of the construct they're declaring the type for 
    
        let age: number = 30;  //Age is explicitly annotated as a number.

Here are some key points about type annotations in TypeScript:

  • They help catch JavaScript errors by enforcing type checking.
  • They can be used to specify the type of variables, function parameters, and object properties.
  • Once an identifier is annotated with a type, it can be used as that type only. If the identifier is used as a different type, the TypeScript compiler will issue an error.
  • Type annotations can be used for primitive data types such as number, string, and boolean.
  • They can also be used for more complex types such as arrays and objects.
  • Type annotations can be used to specify the return type of functions.

Basic Type Annotations for variables:

You can use type annotations when declaring variables to specify their data type.

    
        let name: string = 'Alice';
        let isActive: boolean = true;
        let counter: number = 5;
        const powerLevel: bigint = 9000n + 1n;

You cannot change the value using a different data type other than the declared data type of a variable. If you try to do so, TypeScript compiler will show an error. This helps in catching JavaScript errors.

Type Annotations for function parameters and return values

Type Annotations can be applied to function parameters and return values, ensuring that the function accepts and produces the correct data types. 

    
        function add(a: number, b: number): number {
            return a + b
        }

Here, the parameters must be numbers...and the third "number" word, shows, that return from this function must be a number as well.

In TypeScript, the 'void' type is used to indicate that a function does not return any value. 

    
        function add(a: number, b: number): void {
            a + b //No return statement
        }

If the intention of the function is to perform an operation without returning a value, you can keep the 'void' return type but remove the return statement.

The choice between these options depends on the intended behavior of your function. If the function is meant to perform a calculation and return a result, use type of variable, that should be returned. It it's meant to perform an operation without returning a value, use 'void' return type.

Type Inference

TypeScript can often infer the data type based on the value assigned to a variable. This is known as type inference. 

        
            let temperature = 25; //TypeScript infers that temperature is of type number

However, you can still provide explicit type annotations for clarity and to catch potentail errors.

Union types and type aliases:

TypeScript supports more advance type annotations like 'union types' and type aliases.

  • Union Types: Allow a variable to have multiple possible types. 
        
        let result: number || string = 42;
  • Type Aliases: Let you create custom names for complex types, making your code more readable. 
        

        type myString = string;
        let country: myString = "USA";


        type Point = { x: number; y: number};
        let origin: Point = {x: 0, y: 0}

Mnemonic Devices:

  • Think of type annotations as labels that specify the data type of variables and functions.
  • Union types are like having a variable that can wear multiple hats, and type aliases are like giving names to complex structures for better organization.

Type annotations in TypeScript, a feature that enhances type safety, improves code readability, and provides valuable documentation. Type annotations are crucial for building robust and maintainable TypeScript applications.
