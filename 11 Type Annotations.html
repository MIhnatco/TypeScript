Type Annotations

Type annotations in TypeScript are used to explicitly specify types for variables, functions, objects, and more. They are not mandatory in TypeScript, but they help the compiler in checking types and avoiding errors when dealing with data types.

Type annotations are declared using a colon (:) followed by the name of the type, and they come after the name of the construct they're declaring the type for 
    
        let age: number = 30;  //Age is explicitly annotated as a number.

Here are some key points about type annotations in TypeScript:

  • They help catch JavaScript errors by enforcing type checking.
  • They can be used to specify the type of variables, function parameters, and object properties.
  • Once an identifier is annotated with a type, it can be used as that type only. If the identifier is used as a different type, the TypeScript compiler will issue an error.
  • Type annotations can be used for primitive data types such as number, string, and boolean.
  • They can also be used for more complex types such as arrays and objects.
  • Type annotations can be used to specify the return type of functions.

Basic Type Annotations for variables:

You can use type annotations when declaring variables to specify their data type. 

    
        let name: string = 'Alice';
        let isActive: boolean = true;
        let counter: number = 5;
        const powerLevel: bigint = 9000n + 1n;

You cannot change the value using a different data type other than the declared data type of a variable. If you try to do so, TypeScript compiler will show an error. This helps in catching JavaScript errors.

If you assign a string to the counter variable, you'll get an error: 

    
        let counter: number;
        counter = 'Hello'; // compile error  Type '"Hello"' is not assignable to type 'number'.

Type Annotations for function parameters and return values

Type Annotations can be applied to function parameters and return values, ensuring that the function accepts and produces the correct data types. 

    
        function add(a: number, b: number): number {
            return a + b
        }

Here, the parameters must be numbers...and the third "number" word, shows, that return from this function must be a number as well.
TypeScript will verify that the function is called with the correct types of arguments and will enforce the return type.

In TypeScript, the 'void' type is used to indicate that a function does not return any value. 

    
        function add(a: number, b: number): void {
            a + b //No return statement
        }

If the intention of the function is to perform an operation without returning a value, you can keep the 'void' return type but remove the return statement.

The choice between these options depends on the intended behavior of your function. If the function is meant to perform a calculation and return a result, use type of variable, that should be returned. It it's meant to perform an operation without returning a value, use 'void' return type.

Type Annotations for Arrays

Annotate an array type you use use a specific type followed by a square bracket : type[] : 

    
let arrayName: type[];

For example, the following declares an array of strings:

    
        let names: string[] = ['John', 'Jane', 'Peter', 'David', 'Mary'];

The following declares an array of numbers:

    
        let myArray: number[] = [1, 2, 3];

In the examples above, we have annotated 'names' as an array of strings and 'myArray' as an array of numbers.

Type Annotations for objects.

In TypeScript, you can use type annotations to specify the structure and types of the properties within an object. You can define a type that represents the shape of an object and then annotate the object with that type.

Here's how you can add type annotations for the you object in your code: 

    
        let myObject: { name: string, age: number } = {
            name: "John Doe", 
            age: 24
        }

Type annotations can be used throughout your TypeScript code to provide clarity and catch potential type-related errors. They also help with code documentation and improve tooling support, such as autocompletion and type checking in IDEs.

Type Inference

TypeScript can often infer the data type based on the value assigned to a variable. This means that you don't always have to explicitly annotate the types of variables, as TypeScript can infer them for you. This is known as type inference.

Here's an example:

 
            
            let myNumber = 42; // TypeScript infers the type as number
            let myString = "Hello, TypeScript!"; // TypeScript infers the type as string
            let myBoolean = true; // TypeScript infers the type as boolean

In this case, we haven't provided explicit type annotations for the variables myNumber, myString, and myBoolean. However, TypeScript automatically infers their types based on the assigned values.

Type inference also works with function parameters and return types: 

        
            function add(a: number, b: number) { 
                return a + b; // TypeScript infers the return type as number
             } 
                
            
            let result = add(2, 3); // TypeScript infers the type of result as number

In this example, the function add doesn't have an explicit return type annotation. However, TypeScript is able to infer that the return type is number based on the fact that we are performing addition on two parameters of type number.

Type inference is a handy feature that can help reduce code verbosity and improve code readability. However, you can still provide explicit type annotations for clarity and to catch potentail errors.
